Screen A — User Management: The command center for all users. The Access Source column is critical — it shows HOW the user got their access: Paid (Stripe), Token (Founder/Ambassador/Academic), Free · Admin (manually granted), or Free (registered). Yellow rows = token holders with low engagement (nudge candidates). Blue rows = admin-granted free access. Bulk actions allow mass tier changes, tagging, and export. Impersonate (🎭) lets admins login as the user to see exactly what they see.

Screen B — Grant Free Access: Admins can grant any tier to any user with full audit trail. Reason is mandatory — no anonymous free handouts. The system creates an access_grants row, updates the user's tier, optionally creates an Ambassador token (for referral tracking), and sends a welcome email. A $0 Stripe subscription is created so the user appears in subscription reports. All admin actions are logged in subscription_events with initiated_by='admin'.
